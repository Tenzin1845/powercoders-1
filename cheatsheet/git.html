Git and GitHub Cheatsheet

Git is a Version Control System (VCS).

  • Lets us take a collection of files and save their current contents
  • Saving is called "committing" the changes
    • When we commit the changes we can provide a message that explains why
  • Then we can make changes to the files and:
    • See what's changed (diff)
    • Easily undo the changes (or some of them)
    • Or commit these as new versions of the files
  • Over time this builds up a timeline of changes to the files

Git has 3 location.

  • Working Directory
  • Index
  • Repository

Git commands copy files between these location.

Git Workflow

              Working directory
                    ↓↑
                git diff ...
                    ↓
                git add ...
                    ↓
                  Index
                    ↓
                git commit ...
                    ↓
                Repository
  1. Edit one or more files to make your change
  2. Copy the files to the index, with git add
  3. Review the changes in the index
    • Double check everything is included
    • Remove files not related to this change
  4. Move the files from the index to the repository, with git commit

git config

  • Configure the git User name and user E-mail.
  • % git config --global user.email sumanthini.majutharan@powercoders.org
  • % git config --global user.name "Sumanthini Majutharan"

git init

  • Tell Git where the root of the Repository is.
  • %.....poco>git init
  • Now everything in the poco folder can be managed by git

git status

  • Lists all tracking files to be commited
  • %git status

git add

  • git add will add the files to index.
  • %git add file-name to add single file
  • %git add file1.ext file2.ext file3.ext to add multiple files
  • %git add . to add all the files in the given folder.

git rm

  • git rm will remove the file from index.
  • %git rm --cached file

git commit

  • git commit will move the files from the index to repository.
  • %git commit -m "message"

git log

  • Display commit logs.
  • %git log

git diff

  • This shows the difference between what's in the working directory and what's in the index (or repository).
  • %git diff
  • git diff --cached (or --staged) displays the differnece between the file in index and repository

git mv

  • %git mv old-name new-name to rename the filesb.
  • Don't forget to commit the changes

git rm file-name

  • %git mv file-name to remove the files.
  • Don't forget to commit the changes

GitHub steps

  • Create a GitHub account
  • Create a repository at GitHub
  • Configure your local repository to sync to GitHub repository
  • Push commits to GitHub
  • View commits on GitHub
  • Leave comments on GitHub

Configure your local repository to sync

  • Other copies of your repository are called remotes
  • Find your GitHub repository URL
    • Looks like https://github.com/user/poco-yourname.git
  • %git remote add origin https://github.com/user/poco-yourname.git
  • %git remote -v to view configured remotes

git push

  • %git push to send the changes to GitHub.
  • Run for 1st time : %git push -u origin master
  • Later: %git push
